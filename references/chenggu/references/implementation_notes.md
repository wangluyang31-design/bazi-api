# 实现说明

## 1. 版本锁定
- 本包固定采用 `standard-51qian` 版本。
- 总骨重只在 **21~71 钱** 范围内闭合；不外推，不补写 72 钱，也不混入男女分版断语。

## 2. 输入策略
优先支持两种输入：
1. `birth_datetime`：自然语言或标准日期时间；若为公历，必须先做可靠的农历转换。
2. `lunar_components`：已归一化的农历组件，适合 AI/工具链之间传递，避免重复转换与误差。

## 3. 农历转换要求
- 该 Skill 的查表与结构化部分是确定性的。
- **公历 → 农历** 不应由模型臆测；应使用当前运行环境中可靠的日历库、MCP、已有服务，或让上游先传入 `lunar_components`。
- 若当前运行环境没有可靠转换能力，应明确告知“无法可靠转换”，而不是硬算。

## 4. 边界规则
- 夜子时：23:00~23:59 视为次日。
- 闰月：1~15 按本月，16~30 按下月。
- 年键：统一以 **六十甲子年干支** 为查表键，而非直接用公历年份。

## 5. 结构化输出原则
- 先给机器可读 JSON。
- 再给中文展示层（`display_zh`）。
- `analysis.career / wealth / marriage` 来源于歌诀规则抽取与人工覆写，不是统计预测模型。
- `trajectory_tags` 用于记录轨迹型特征，例如 `early_hardship`、`midlate_better`、`mobility`、`scholar_official` 等。

## 6. 不要做的事
- 不要脱离查表数据自创权重。
- 不要对没有对应档位的总骨重进行插值。
- 不要把称骨结果包装成科学结论或人生决策依据。
